// Qué incluye

Un set curado, no un cajón de sastre.

Cada skill fue elegida a mano. Tu agente las carga solo cuando la tarea lo pide.

  • Diseño y buen gusto

    Dirección visual, jerarquía y detalle que no parece template.

  • Accesibilidad (WCAG)

    Auditorías de contraste, foco y semántica según pautas WCAG.

  • Motion y animación

    Timing, easing y los 12 principios aplicados a la web.

  • Best practices de React

    Patrones de performance, hooks y diagnóstico de componentes.

  • Componentes shadcn

    Uso correcto del registro, composición y primitivas accesibles.

  • Testing en navegador

    El agente abre el navegador y verifica la UI que construyó.

  • Brainstorming

    Explora direcciones antes de escribir una sola línea de UI.

  • + más, todas opt-in
    y cargadas bajo demanda

// Cómo funciona

Tres pasos. Cero configuración.

  1. 01

    Corrés el comando

    Un curl … | bash (o irm … | iex en Windows). No instala nada global ni toca tu PATH.

  2. 02

    Las skills bajan a disco

    Se copian a ~/.claude/skills. Son archivos de texto: leelas, editalas o borralas cuando quieras.

  3. 03

    Tu agente las usa solo

    Cuando una tarea de UI lo amerita, el agente carga la skill relevante. Sin que se lo recuerdes.

// Instalación

Elegí tu forma.

$ curl -fsSL https://raw.githubusercontent.com/juampymdd/skills-install/main/install-ui-skills.sh | bash
> irm https://raw.githubusercontent.com/juampymdd/skills-install/main/install-ui-skills.ps1 | iex
con npx (Node)
$ npx github:juampymdd/skills-install
con bunx (Bun)
$ bunx github:juampymdd/skills-install

Internamente las skills se bajan con la CLI skills (Node), así que necesitás Node o Bun instalado.
